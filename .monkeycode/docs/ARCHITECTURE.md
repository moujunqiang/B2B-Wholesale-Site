# 系统架构设计

## 整体架构

本项目采用 Serverless 架构，基于 Cloudflare Workers 平台构建，实现前后端分离的 B2B 外贸独立站。

## 技术选型

### 核心框架
- **Hono**: 轻量级 Web 框架，专为边缘计算设计
  - 优秀的 TypeScript 支持
  - 内置常用中间件（CORS, Logger, Basic Auth 等）
  - 与 Cloudflare Workers 完美集成

### 数据存储
- **Cloudflare D1**: 基于 SQLite 的边缘数据库
  - 低延迟读取
  - SQL 语法兼容性好
  - 适合结构化数据存储
  
- **Cloudflare R2**: 对象存储服务
  - 零出口费用
  - 适合存储产品图片等静态资源
  - 与 Workers 无缝集成

### 类型系统
- **TypeScript**: 提供类型安全和开发体验
- **Zod**: 运行时类型校验，用于 API 请求验证

### 认证方案
- **HTTP Basic Authentication**: 简单有效的后台认证方式
  - 适合小型后台管理系统
  - 配合 HTTPS 使用
  - 易于实现和维护

### 主要表结构

#### products (产品表)
- 核心业务表，存储产品信息
- 关联 category_id 到分类表
- 支持图片数组和规格字典（JSON 格式）

#### categories (分类表)
- 树形结构，支持多级分类
- parent_id 自引用实现层级关系
- sort_order 控制显示顺序

#### inquiries (询盘表)
- 客户询盘记录
- status 字段跟踪处理状态
- 关联 product_id 记录感兴趣的产品

#### translations (翻译表)
- 多语言支持
- locale + key 唯一索引
- 支持动态添加语言

## 核心流程

### 1. 用户浏览产品流程

```
用户请求 → CDN → Workers → 查询 D1 → 返回产品数据 → 渲染页面
```

### 2. 提交询盘流程

```
用户填写表单 → 前端验证 → API 请求 → 后端验证 → 
写入 D1 → 返回成功 → 前端提示 → (可选) 发送邮件通知
```

### 3. 后台管理流程

```
管理员登录 → Basic Auth → 验证通过 → 访问后台 API → 
CRUD 操作 → 更新 D1 → 返回结果
```

## 安全设计

### 认证
- 后台 API 使用 HTTP Basic Authentication
- 密码采用 PBKDF2 + SHA256 哈希存储
- 建议生产环境开启 HTTPS

### 输入验证
- 使用 Zod 进行请求体验证
- SQL 参数化查询防止注入
- XSS 防护（Hono 默认）

### 数据安全
- 敏感数据（密码）加密存储
- 最小权限原则
- 定期备份 D1 数据

## 性能优化

### 边缘计算优势
- Workers 部署在全球边缘节点
- 就近访问，降低延迟
- D1 数据自动复制

### 缓存策略
- 静态资源通过 CDN 缓存
- 产品列表可添加缓存层
- 设置合理的 Cache-Control 头

### 数据库优化
- 关键字段建立索引
- 分页查询避免大数据量
- 定期清理历史数据

## 扩展性设计

### 水平扩展
- Workers 自动扩展
- D1 支持多数据库实例
- R2 无限存储扩展

### 功能扩展
- 模块化 API 设计
- 易于添加新业务模块
- 支持多语言动态扩展

### 集成扩展
- RESTful API 设计
- 易于对接第三方系统
- 支持 Webhook 扩展

## 部署架构

```bash
npm run dev  # 本地开发服务器
wrangler d1 execute --local  # 本地 D1
```

### 生产环境
```bash
npm run deploy  # 部署到 Cloudflare
wrangler d1 execute --remote  # 远程数据库操作
```
